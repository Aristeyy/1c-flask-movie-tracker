# 🎬 КиноДашборд

Связка **1С:Предприятие** + **Flask** для учёта просмотренных фильмов: конфигурация 1С служит фронтендом для ввода данных, а веб-дашборд на Flask — витриной со статистикой, графиками и обогащением данных из OMDB.

## Как это устроено

- В **1С:Предприятие** есть конфигурация «КинДашборд»: справочник **Фильмы** (со статусами «хочу посмотреть» / «смотрю» / «просмотрено» / «брошено») и документ **Просмотр фильма**, в котором фиксируется дата просмотра, оценка и эмоция.
- При проведении документа 1С отправляет данные просмотра HTTP-запросом (`POST /api/viewing`) во Flask-сервер.
- Flask сохраняет просмотр в SQLite и показывает его на веб-дашборде: статистика, тепловая карта активности, графики по жанрам и оценкам, рекомендации, подтягивание описания/постера/трейлера из OMDB.
- Дашборд также работает автономно — фильм можно добавить, отредактировать и удалить прямо с сайта, без 1С.

### Информационная база 1С

В файле [`kinodashboard.dt`](./kinodashboard.dt) лежит выгрузка информационной базы 1С (конфигурация + данные). Чтобы открыть её:

1. Запустите **1С:Предприятие** → **Добавить** информационную базу → **Создание новой информационной базы** → **Загрузка из файла**.
2. Укажите путь к `kinodashboard.dt`.
3. После создания базы откройте её в режиме «1С:Предприятие» — конфигурация уже настроена на отправку данных в Flask на `http://localhost:5000`.

## Возможности веб-дашборда

- Тёмная/светлая тема, кинематографичный UI с собственной типографикой
- Добавление, редактирование и удаление просмотров
- Загрузка своего постера к фильму, личные заметки
- Подтягивание описания, постера, рейтинга IMDb и трейлера из [OMDB](https://www.omdbapi.com/) с кэшированием в БД
- Статистика: средний рейтинг, разбивка по жанрам/эмоциям, тепловая карта активности по дням недели
- Графики (Chart.js): распределение по жанрам, динамика оценок
- Рекомендации на основе любимого жанра
- Импорт/экспорт в CSV, бэкап в JSON
- Приём данных от 1С через REST API

## Установка и запуск

```bash
git clone https://github.com/<your-username>/kinodashboard.git
cd kinodashboard

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

Создайте файл `.env` на основе `.env.example` и впишите свой ключ OMDB (бесплатно на [omdbapi.com](https://www.omdbapi.com/apikey.aspx)):

```bash
cp .env.example .env
```

```
OMDB_API_KEY=ваш_ключ
```

Запуск:

```bash
python app.py
```

Дашборд будет доступен на [http://127.0.0.1:5000](http://127.0.0.1:5000). База данных `movies.db` создаётся автоматически при первом запуске.

Информационную базу 1С (`kinodashboard.dt`) при желании можно загрузить отдельно — см. раздел выше.

## API

| Метод  | Путь                        | Описание                              |
|--------|-----------------------------|----------------------------------------|
| POST   | `/api/viewing`              | приём просмотра от 1С                  |
| GET    | `/api/movies`                | список просмотров                      |
| POST   | `/api/movies`                | добавить фильм вручную                 |
| PATCH  | `/api/movies/<id>`           | изменить запись                        |
| DELETE | `/api/movies/<id>`           | удалить запись                         |
| POST   | `/api/movies/<id>/poster`    | загрузить постер                       |
| GET    | `/api/stats`                 | агрегированная статистика              |
| GET    | `/api/recommend`             | рекомендации по жанру                  |
| GET    | `/api/fill?name=...`         | данные из OMDB (с кэшем)               |
| GET    | `/api/export/csv`            | экспорт в CSV                          |
| GET    | `/api/export/json`           | бэкап в JSON                           |
| POST   | `/api/import/csv`            | импорт просмотров из CSV               |
